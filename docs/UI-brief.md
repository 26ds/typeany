# UI 需求书(直接整段复制给 GPT/任何 AI 设计顾问)

```text
你是资深 UI/视觉设计顾问。我在做一个打字练习网站(开发代号 TypeAny),基于 monkeytype 的开源前端 fork,请帮我确定完整视觉方案。

【产品背景】
- 参考 monkeytype.com 的极简暗色打字体验:打字区无边框、大字号等宽字体、彩色光标动画、顶部胶囊形配置条
- 核心差异:用户可上传自己的书(PDF/EPUB/TXT,中英文),按章节顺序或随机段落练打字,边打字边读书
- 设计基调已定死:全站玻璃拟态 glassmorphism(毛玻璃半透明卡片+按钮+背景),暗色为主,弹窗出现时背景整体模糊

【页面清单(都要设计)】
1. Landing:科技感字体大 logo + 两个入口按钮「Upload & Type」「Random Mode」
2. Random 打字页:monkeytype 式。顶栏:logo/键盘图标/设置/账号;配置条:punctuation、numbers | time、words、custom | 10/25/50/100
3. 书籍打字页:打字区同上;左上角两条「可点击」的细进度条(本书已打%/本章已打%);屏幕左右侧「上一块/下一块」箭头;右下角两个结算按钮:「结算(计入个人数据和历史)」和「结算(当前轮不计入)——配小一号注释文字」
4. 选章弹窗:全书目录(章节名列表),显示各章进度
5. 选页弹窗(重点):背景毛玻璃全模糊,半屏大弹窗。左侧=仿原书分页视图(图片位置留白;文字可鼠标选中;已打过的文字按打过次数由浅到深着色,没打过=灰色);顶部「直达 page:__」输入+左右翻页箭头,底部页码;中间一列热力图比色卡;右侧=选中文本的实时打字预览框;右下角「开始」按钮
6. 结算页:wpm、acc、raw、consistency、characters、time + 每秒速度曲线图;书籍模式加书名/章节/位置和 next 按钮
7. 书架页(已上传的书列表)、上传流程页

【关键显示状态】
- 打字文本三态:未打=灰、打对=亮色、打错=红;光标平滑移动动画
- 自动跳过的字符(数学符号、公式等不用打的)要弱化灰显,和"未打"区分开
- 界面必须同时支持中英文排版不崩

【要你产出】
1. 2-3 套完整视觉方向,每套含:背景处理(渐变/噪点/光斑?)、玻璃参数(blur 值/透明度/边框/阴影)、色板 hex(底色+主色+强调色+错误红)、字体配对(landing 科技感展示字体 + 打字区等宽字体,给出含中文兜底的 font-family 列表)、圆角与间距体系
2. 每个页面的布局线框描述(文字版即可),重点抠「选页弹窗」和「书籍打字页」
3. 热力图 6-8 档渐变色值(GitHub 贡献图思路但档位更多、更有渐变感)
4. 动效建议:光标、按钮 hover、弹窗模糊过渡
5. 对比度过 WCAG AA(暗色玻璃上的文字易翻车,给我能过检的组合)
先给 2-3 套方向概览让我选,我选定后你再逐页细化。过程中信息不够就问我。
```
