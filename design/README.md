# TypeAny 视觉方案

> 设计阶段：视觉方向概览
>
> 本文档用于 TypeAny（开发代号）的首轮视觉方向决策。当前版本先确定整体基调；选定方向后，再补充逐页线框、组件状态和交互细节。

## 1. 产品视觉基线

TypeAny 是一个“边读书边打字”的练习产品。视觉上继承极简暗色打字体验，但需要通过书籍、章节、分页和阅读状态，建立区别于普通速度测试工具的产品识别度。

三套方向都遵循以下固定原则：

- 全站使用暗色 glassmorphism：半透明面板、背景模糊、低对比度边框和柔和阴影。
- 打字区保持无边框、低干扰、大字号等宽字体。
- 未打、打对、打错、自动跳过字符必须有清晰且稳定的视觉区分。
- 中文和英文混排时，光标、错误标记和字符槽位不能漂移。
- 弹窗出现时，背景整体变暗并模糊，阅读焦点转移到弹窗内部。
- 书籍功能比 Random Mode 多一层“阅读”和“进度”信息，但不能抢过当前打字内容。

## 2. 三套方向总览

| 方向 | 气质 | 优点 | 风险 | 适合场景 |
|---|---|---|---|---|
| A. Neon Graphite | 霓虹石墨、科技、速度 | 最接近 monkeytype，游戏化反馈强 | 容易与其他霓虹工具同质化 | Random Mode、挑战、速度数据 |
| B. Ink Aurora | 墨迹极光、数字书房、阅读 | 最能体现“书籍 + 打字”的差异化 | 需要控制装饰，避免变成阅读器 | 书架、选章、选页、书籍打字 |
| C. Obsidian Lime | 黑曜石、荧光、专业 | 对比度高、耐看、数据清晰 | 情绪和品牌记忆点稍弱 | 长时间练习、统计、历史数据 |

初步推荐：以 B 为主视觉，吸收 C 的高对比度和 A 的光标动效。

## 3. 方向 A：Neon Graphite｜霓虹石墨

最接近 monkeytype 的基础体验，但增加蓝紫色光晕、数据高亮和科技感 Logo。

### 背景

使用深石墨背景、低强度径向光斑、细网格和 2%～4% 噪点。

```css
background:
  radial-gradient(circle at 20% 10%, rgba(65, 180, 255, .14), transparent 34%),
  radial-gradient(circle at 85% 80%, rgba(145, 95, 255, .12), transparent 32%),
  #080B12;
```

### 玻璃参数

- 卡片：`rgba(20, 25, 38, .58)`
- 弹窗：`rgba(16, 21, 33, .72)`
- 毛玻璃：`backdrop-filter: blur(24px) saturate(135%)`
- 边框：`1px solid rgba(255,255,255,.12)`
- 顶部内高光：`rgba(255,255,255,.08)`，1px 渐变
- 阴影：`0 16px 48px rgba(0,0,0,.34)`
- 弹窗遮罩：`rgba(3,6,12,.52)` + `blur(18px)`

### 色板

| 用途 | 色值 |
|---|---|
| 页面底色 | `#080B12` |
| 卡片底色 | `#141926` |
| 主色：正确文字、主按钮 | `#65E6FF` |
| 强调色：光标、选中态 | `#9B7CFF` |
| 辅助色：书籍进度 | `#68D5FF` |
| 错误红 | `#FF5F73` |
| 主文字 | `#F5F7FA` |
| 次文字 | `#AAB4C3` |
| 弱文字 | `#7F899B` |
| 自动跳过字符 | `#4B5566` |

### 字体

Landing 和 Logo：

```css
font-family: "Space Grotesk", "Noto Sans SC", "PingFang SC",
             "Microsoft YaHei", sans-serif;
```

Logo 可使用 `Orbitron` 作为英文字标：

```css
font-family: "Orbitron", "Space Grotesk", "Noto Sans SC", sans-serif;
```

打字区：

```css
font-family: "JetBrains Mono", "Noto Sans Mono CJK SC",
             "Noto Sans SC", "PingFang SC", monospace;
```

### 圆角与间距

- 小按钮：`10px`
- 配置条：`16px`
- 大卡片、弹窗：`24px`
- 页面主间距：`24px`
- 组件间距：`12px`
- 基础间距单位：`4px`

## 4. 方向 B：Ink Aurora｜墨迹极光

更像“数字书房 + 打字练习工具”。不追求强烈霓虹，而是使用墨绿黑、青绿色和琥珀色，强化阅读和纸张的联想。

### 背景

背景以墨绿黑为主，加入轻微纸张纤维噪点和模糊光晕。

```css
background:
  radial-gradient(circle at 15% 20%, rgba(75, 170, 135, .13), transparent 32%),
  radial-gradient(circle at 82% 15%, rgba(255, 180, 85, .10), transparent 28%),
  #0B1112;
```

选页弹窗中的原书预览可使用极低强度的纸张色：`rgba(221, 211, 184, .055)`，不要使用纯白页面。

### 玻璃参数

- 卡片：`rgba(29, 42, 40, .62)`
- 弹窗：`rgba(18, 29, 29, .78)`
- 毛玻璃：`backdrop-filter: blur(30px) saturate(120%)`
- 边框：`1px solid rgba(190, 235, 215, .14)`
- 阴影：`0 20px 60px rgba(0,0,0,.38)`
- 弹窗遮罩：`rgba(3,10,10,.58)` + `blur(24px)`

## 5. 方向 C：Obsidian Lime｜黑曜石荧光

更克制、更偏专业效率工具。把视觉重点集中在文本、光标、统计曲线和热力图，适合长时间练习。

### 背景

近乎黑色的黑曜石背景，仅保留非常低强度的顶部荧光和噪点。

```css
background:
  radial-gradient(circle at 50% -10%, rgba(210, 255, 100, .07), transparent 35%),
  #090C0E;
```

### 玻璃参数

- 卡片：`rgba(25, 29, 31, .76)`
- 弹窗：`rgba(17, 21, 23, .86)`
- 毛玻璃：`backdrop-filter: blur(18px) saturate(110%)`
- 边框：`1px solid rgba(255,255,255,.14)`
- 阴影：`0 12px 36px rgba(0,0,0,.42)`
- 弹窗遮罩：`rgba(0,0,0,.64)` + `blur(20px)`

## 6. 统一热力图色阶

建议使用“冷灰 → 蓝青 → 绿色 → 黄绿”的 8 档渐变，延续 GitHub 贡献图逻辑，但比单纯绿色有更好的层次感。

| 档位 | 含义 | 色值 |
|---|---|---|
| 0 | 未打过 | `#3E4650` |
| 1 | 打过 1 次 | `#28454E` |
| 2 | 低频 | `#206473` |
| 3 | 中低频 | `#218B91` |
| 4 | 中频 | `#27B39D` |
| 5 | 高频 | `#54C96D` |
| 6 | 很高频 | `#A5E35D` |
| 7 | 最高频 | `#F1EF82` |

自动跳过字符不使用热力图颜色，始终保持灰色弱化，避免用户误以为这些字符拥有练习数据。

## 7. 统一文本状态

- 未打：使用对应方向的弱文字色。
- 打对：使用主色。
- 打错：使用错误红，并配合波浪线、底部标记或轻微抖动，不能只依赖颜色。
- 自动跳过：使用更暗的灰色和较低透明度，与“未打”保持明显区别。
- 当前字符：使用主色背景或 2px 光标，不使用红色边框。

建议把光标渲染为独立动画层：宽度 `2px`，高度为当前行高的 `1.15` 倍，使用 `120～180ms cubic-bezier` 平滑移动；闪烁时保留约 `35%` 不透明度，不要完全消失。

中英文混排不要只依赖 CSS 的 `monospace`。建议按 grapheme cluster 拆分文本，并为 Latin、数字、中文分别分配稳定字符槽位，避免中文宽度导致光标和错误标记漂移。

## 8. 动效基线

### 光标

- 光标位置使用 `transform: translateX()`，避免改变布局。
- 字符切换时 `120～180ms`，快速连续输入时可缩短到 `80ms`。
- 书籍模式切换段落时，光标先淡出 `80ms`，新位置淡入 `120ms`。

### 按钮

- Hover：亮度增加 8%～12%，上移 `1px`，过渡 `140ms`。
- Press：缩放到 `scale(.98)`，不做明显弹跳。
- 主按钮可增加低强度主色外发光，但不要超过按钮本身亮度。

### 弹窗

- 背景遮罩和模糊：`180～260ms ease-out`。
- 弹窗：`translateY(10px) scale(.985)` 到正常状态，`220ms cubic-bezier`。
- 关闭时先降低内容透明度，再移除模糊，避免背景突然跳变。
- `prefers-reduced-motion: reduce` 时关闭位移动画，只保留透明度变化。

## 9. WCAG AA 基线

暗色玻璃的对比度必须按“最差背景透出状态”检查，不能只测试设计稿中的单一底色。

- 正常正文目标对比度：`≥ 4.5:1`。
- 大标题、按钮文字目标对比度：`≥ 3:1`。
- 主文字建议使用 `#F3F5F0`、`#EEF8F1` 或 `#F5F7FA`。
- 正文次文字建议至少使用 `#AAB4C3`、`#B3C3BB` 或 `#B5BFBA`。
- 不要使用低于 `#7F899B` 的灰色作为正文或关键提示。
- 霓虹色用于大字号、按钮、光标和状态指示，不用于小字号说明文字。
- 错误红使用 `#FF5F73`、`#FF6B6B` 或 `#FF6B78`，同时配合形状/纹理标记。
- 玻璃面板底部增加 `rgba(5,8,10,.68)` 左右的暗色底，避免文字因背景光斑失去对比度。
- 小字号注释文字放在透明度更高、背景更稳定的面板区域。

C 方向最容易通过 AA；A 方向科技感最强；B 方向最符合 TypeAny 的书籍产品定位。

## 10. 下一步

视觉方向确定后，继续细化：

1. Landing、Random Mode、书籍打字页、选章弹窗、选页弹窗、结算页、书架页和上传流程的文字版线框。
2. 选页弹窗的三栏尺寸、分页区域、热力图图例、实时预览和开始按钮状态。
3. 书籍打字页的双进度条、左右切块导航、结算按钮层级和键盘快捷键。
4. 组件 token、响应式断点、中文/英文混排规则和无障碍交互状态。
